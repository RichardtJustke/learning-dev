## Roadmap (em construção)

- Fase 00 — Fundação CS (ABR–JUN, ano 1)
	- Foco: entender como o computador, rede e sistema operacional funcionam na prática.
	- LeetCode: Easy (arrays, strings, hash maps), 2-3x por semana.
	- Integração: ainda sem stack completa, foco total em base técnica.
	- Projetos-chave:
		- Ambiente Linux do zero
		- Estruturas de dados do zero em Go
		- Algoritmos com benchmark
		- Clone simples do curl
		- Leitor de processos do SO
		- Banco chave-valor em arquivo
	- Marco: explicar com clareza o caminho completo de uma URL até a resposta no navegador.

- Fase 01 — Go Raiz (JUL–AGO, ano 1)
	- Foco: escrever Go idiomático sem depender de atalhos.
	- LeetCode: Easy/Medium (sliding window, two pointers, maps).
	- Integração: Go puro, sem framework e sem ORM.
	- Projetos-chave:
		- HTTP server sem framework
		- Grep clone
		- Sistema de arquivos com locking
		- LRU cache genérico com TTL
		- Mini ORM com reflection
		- Worker pool com errgroup
	- Marco: construir lógica em Go com fluidez e domínio da linguagem.

- Fase 02 — Backend Real (SET–OUT, ano 1)
	- Foco: backend de produção com Go + PostgreSQL + TypeScript.
	- LeetCode: Medium (trees, graphs e DP básico), 4x por semana.
	- Integração: Go backend ↔ PostgreSQL ↔ TypeScript frontend.
	- Projetos-chave:
		- API de autenticação completa (JWT + refresh)
		- Migrations e modelagem relacional real
		- Contratos tipados via OpenAPI
		- Sistema de upload com processamento assíncrono
		- Rate limiter com Redis
		- Testes de integração ponta a ponta
	- Marco: manter uma API em produção sendo consumida por cliente TypeScript.

- Fase 03 — Performance & Internals (NOV–JAN, ano 2)
	- Foco: profiling, gargalos e otimização guiada por dados.
	- LeetCode: Medium/Hard (heap, trie, graph traversal), 4-5x por semana.
	- Integração: Go com medição real antes de qualquer otimização.
	- Projetos-chave:
		- Profiling com pprof
		- Connection pool do zero
		- Crawler com backpressure
		- Chat realtime com WebSocket
		- Benchmark comparativo de abordagens
		- Mini banco em disco (B-tree + WAL)
	- Marco: identificar e resolver gargalos reais com evidência técnica.

- Fase 04 — Distribuição & Escala (FEV–MAI, ano 2)
	- Foco: microserviços, mensageria e resiliência.
	- LeetCode: Hard + preparação de System Design.
	- Integração: Go services ↔ gRPC ↔ TypeScript client ↔ Kafka ↔ PostgreSQL.
	- Projetos-chave:
		- Microserviço com gRPC
		- Cache em camadas (L1 + Redis)
		- Pipeline com Kafka + DLQ
		- Distributed lock
		- Circuit breaker do zero
		- Event sourcing
	- Marco: projetar sistema para 1 milhão de usuários justificando trade-offs.

- Fase 05 — Big Tech Stack (JUN–DEZ+, ano 2+)
	- Foco: plataforma de produção completa com cloud-native e observabilidade.
	- LeetCode: Hard + mock interviews + system design semanal.
	- Integração: Go + TS + PG + Kafka + K8s + OpenTelemetry.
	- Projetos-chave:
		- Deploy com Kubernetes
		- Observabilidade distribuída (OpenTelemetry + Jaeger + Grafana)
		- Documentos de system design
		- Contribuição real em open source
		- SaaS completo em produção
	- Marco: estar pronto para entrevistas técnicas de Big Tech.

- Metas de carreira
	- 12–18 meses: Júnior/Pleno (Go em produto real)
	- 24–30 meses: Sênior (sistemas distribuídos e decisões técnicas)
	- 36+ meses: Big Tech/Staff (escala real e impacto técnico)
